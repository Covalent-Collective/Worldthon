# Seed Vault - 데모 시나리오

## 개요

- **소요 시간**: 3분
- **대상**: 해커톤 심사위원
- **목표**: World ID 기반 음성 저널링 + 지식 기여/보상 시스템의 핵심 가치 전달

---

## 오프닝 (30초)

### 문제 제기
> "2026년, 인터넷의 90%가 AI가 생성한 콘텐츠입니다.
> 진짜 인간의 경험과 지식은 어디서 찾을 수 있을까요?"

### 솔루션 소개
> "Seed Vault는 World ID Orb로 인증된 인간만 기여할 수 있는
> 지식 저장소입니다. 음성으로 기록하면 AI가 키워드를 추출하고,
> 적합한 Vault에 기여됩니다. 기여한 지식이 인용될 때마다 보상을 받습니다."

---

## 데모 Step 1: 저널링 & 기여 (60초)

### 화면
랜딩 페이지 (`/`) -> 저널링 홈 (`/` 인증 후)

### 액션
1. 앱 첫 화면 보여주기 (다크 글래스모피즘 디자인, AuroraBackground)
2. 하단 통계 확인: 노드 수, 기여자 수, Vault 수
3. "World ID로 시작하기" 버튼 클릭 -> 인증 완료
4. 저널링 홈 화면: VoiceOrb (3D CSS sphere) 확인
5. VoiceOrb 탭 -> 녹음 시작 (오브 변형, MM:SS 타이머 표시)
6. 몇 초 후 다시 탭 -> "AI가 분석 중입니다..." 처리 화면
7. 분석 완료: 추출된 키워드 (감정/토픽 태그) 확인
8. RECOMMENDED VAULT 에서 기여할 Vault 선택 (체크박스로 다중 선택)
9. "기여하기" 버튼 클릭 -> "기여 완료!" 화면 + 획득 WLD 표시

### 스크립트
> "World ID로 로그인합니다. 인증된 인간만 접근할 수 있습니다.
> [VoiceOrb 탭]
> 음성으로 내 경험을 기록합니다. 오늘 을지로에서 발견한 숨은 카페에 대해 말해보겠습니다.
> [몇 초 녹음 후 탭]
> AI가 음성을 분석하고 키워드를 추출합니다.
> '을지로', '카페', '분위기' 같은 키워드가 자동으로 태깅되었습니다.
> 추천 Vault 중 '서울 로컬 가이드'를 선택하고 기여합니다.
> [기여하기]
> 기여 완료. WLD 보상이 적립되었습니다."

---

## 데모 Step 2: 지식 그래프 탐색 (45초)

### 화면
Explore 목록 (`/explore`) -> 봇 상세 (`/explore/seoul-local-guide`)

### 액션
1. 하단 네비게이션에서 "Explore" 탭 클릭
2. 3D Carousel에서 봇 카드들 확인
3. "서울 로컬 가이드" 봇의 "탐색하기" 버튼 클릭
4. 지식 그래프 시각화 (Canvas 기반) 확인
5. 통계: 노드, 연결, 인용 수 확인
6. Top Contributors 바 그래프 확인
7. "숨은 맛집 추천해줘" 질문 입력
8. 검색 애니메이션 -> 관련 노드 발견 -> 타이핑 답변 + 기여 영수증

### 스크립트
> "Explore 탭에서 다양한 지식 Vault를 둘러볼 수 있습니다.
> [서울 로컬 가이드 진입]
> 지식 그래프가 시각화되어 있습니다. 현재 노드, 연결, 인용 수를 확인할 수 있습니다.
> [질문 입력]
> '숨은 맛집 추천해줘'라고 질문하면, 그래프에서 관련 노드를 찾아 답변합니다.
> 아래 기여 영수증을 보세요. 이 답변에 사용된 노드와 기여자를 투명하게 보여줍니다.
> 각 기여자는 인용 비율만큼 WLD 보상을 받게 됩니다."

---

## 데모 Step 3: 보상 확인 (30초)

### 화면
보상 대시보드 (`/rewards`)

### 액션
1. 하단 네비게이션에서 "Reward" 탭 클릭
2. Power Level (LV.4) 및 Contribution 37% 확인
3. WLD 카운터 확인 (6.666666 WLD)
4. 통계 그리드: Citations 128, Nodes 3
5. Transaction Log 확인 (최신순 3건)
6. "CLAIM REWARD" 버튼 클릭

### 스크립트
> "보상 대시보드입니다. 현재 Power Level 4이고 Contribution 37%입니다.
> 6.67 WLD가 적립되어 있습니다.
> [CLAIM REWARD 클릭]
> 이렇게 기여하면 할수록 Power Level이 올라가고, 더 많은 보상을 받게 됩니다."

## 예상 질문 & 답변

### 1. "실제로 녹음이 되나요?"
> "데모에서는 시뮬레이션이고, 프로덕션에서는 실제 음성을 텍스트로 변환합니다. Web Speech API 또는 Whisper 기반으로 구현할 계획입니다."

### 2. "실제 WLD가 전송되나요?"
> "MVP에서는 UI만 구현했고, 메인넷 연동은 다음 단계입니다."

### 3. "AI 생성 콘텐츠를 어떻게 구분하나요?"
> "Orb 인증으로 기여자가 인간임을 보장합니다. 콘텐츠 자체의 AI 생성 여부는 커뮤니티 검증으로 보완할 계획입니다."

### 4. "데이터는 어디에 저장되나요?"
> "MVP는 로컬 상태이고, 프로덕션에서는 분산 저장소(IPFS 등)를 고려하고 있습니다."

### 5. "백엔드 아키텍처는 어떻게 되나요?"
> "API 계약서와 데이터베이스 스키마를 문서화해 두었습니다. PostgreSQL + Prisma 기반으로 확장할 계획이며, World ID 검증은 반드시 서버 사이드에서 처리됩니다."

### 6. "nullifier_hash는 어떻게 관리하나요?"
> "익명 식별자로, 개인정보 없이 중복 검증과 기여 추적이 가능합니다. 데이터베이스의 Primary Key로 사용될 예정입니다."

### 7. "90% 통계의 출처는?"
> "Europol 2024 보고서 'Facing Reality'에서 2026년까지 온라인 콘텐츠의 90%가 AI 생성일 수 있다고 예측했습니다."

### 8. "AI 키워드 추출은 어떤 모델을 사용하나요?"
> "데모에서는 사전 정의된 키워드 풀에서 추출하고, 프로덕션에서는 LLM 기반 분석을 적용할 계획입니다."
