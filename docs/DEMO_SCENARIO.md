# Seed Vault - 데모 시나리오

## 개요

- **소요 시간**: 3분
- **대상**: 해커톤 심사위원
- **목표**: World ID 기반 지식 기여/보상 시스템의 핵심 가치 전달

---

## 오프닝 (30초)

### 문제 제기
> "2026년, 인터넷의 90%가 AI가 생성한 콘텐츠입니다.
> 진짜 인간의 경험과 지식은 어디서 찾을 수 있을까요?"

### 솔루션 소개
> "Seed Vault는 World ID Orb로 인증된 인간만 기여할 수 있는
> 지식 저장소입니다. 기여한 지식이 인용될 때마다 보상을 받습니다."

---

## 데모 Step 1: 마켓플레이스 (30초)

### 화면
홈 화면 (`/`)

### 액션
1. 앱 첫 화면 보여주기
2. "서울 로컬 가이드" 봇 카드 확인
3. 노드 수, 기여자 수 설명

### 스크립트
> "이것은 서울 로컬 가이드 봇입니다.
> 12명의 Orb 인증된 서울 시민이 24개의 지식 노드를 기여했습니다."

---

## 데모 Step 2: 검색자 모드 (45초)

### 화면
그래프 탐색 (`/explore/seoul-local-guide`)

### 액션
1. "탐색하기" 버튼 클릭
2. "숨은 맛집 추천해줘" 질문 입력
3. 답변 확인
4. 그래프에서 하이라이트된 노드 보여주기
5. 기여 영수증 설명

### 스크립트
> "질문을 하면, 지식 그래프에서 관련 노드를 찾아 답변합니다.
> 아래 기여 영수증을 보세요. 이 답변은 을지로 골목 맛집 노드 60%,
> 성수동 카페거리 노드 40%로 구성되었습니다.
> 각 기여자는 인용 비율만큼 WLD 보상을 받게 됩니다."

---

## 데모 Step 3: 기여자 모드 (45초)

### 화면
지식 기여 (`/contribute/seoul-local-guide`)

### 액션
1. "기여하기" 버튼 클릭
2. World ID 인증 버튼 클릭
3. (World App에서 Orb 인증)
4. 인증 완료 확인
5. 지식 입력: "망리단길 카페 추천"
6. 제출 → 성공 애니메이션

### 스크립트
> "지식을 기여하려면 먼저 Orb 인증이 필요합니다.
> [인증 진행]
> 인증이 완료되었습니다. 이제 제가 아는 정보를 입력합니다.
> [입력 및 제출]
> 제 지식이 Seed Vault에 영구 보존되었습니다."

---

## 데모 Step 4: 보상 확인 (30초)

### 화면
보상 대시보드 (`/rewards`)

### 액션
1. 하단 네비게이션에서 "보상" 탭 클릭
2. Contribution Power 게이지 확인
3. 총 인용 횟수 확인
4. 수령 가능한 WLD 확인

### 스크립트
> "보상 대시보드입니다. 제 Contribution Power는 15%이고,
> 지금까지 23번 인용되어 0.023 WLD를 받을 수 있습니다.
> 더 좋은 지식을 기여할수록 더 많은 보상을 받게 됩니다."

---

## 클로징 (30초)

### 핵심 메시지
> "Seed Vault는 세 가지 문제를 해결합니다:
>
> 1. **신뢰**: Orb 인증으로 봇과 AI 콘텐츠를 차단합니다
> 2. **인센티브**: 기여에 대한 공정한 보상을 제공합니다
> 3. **투명성**: 답변의 출처를 그래프로 보여줍니다
>
> Dead Internet 시대, Seed Vault가 진짜 인간 지식의 방주가 됩니다."

## 예상 질문 & 답변

### 1. "실제 WLD가 전송되나요?"
> "MVP에서는 UI만 구현했고, 메인넷 연동은 다음 단계입니다."

### 2. "AI 생성 콘텐츠를 어떻게 구분하나요?"
> "Orb 인증으로 기여자가 인간임을 보장합니다. 콘텐츠 자체의 AI 생성 여부는 커뮤니티 검증으로 보완할 계획입니다."

### 3. "데이터는 어디에 저장되나요?"
> "MVP는 로컬 상태이고, 프로덕션에서는 분산 저장소(IPFS 등)를 고려하고 있습니다."

### 4. "백엔드 아키텍처는 어떻게 되나요?"
> "API 계약서와 데이터베이스 스키마를 문서화해 두었습니다. PostgreSQL + Prisma 기반으로 확장할 계획이며, World ID 검증은 반드시 서버 사이드에서 처리됩니다."

### 5. "nullifier_hash는 어떻게 관리하나요?"
> "익명 식별자로, 개인정보 없이 중복 검증과 기여 추적이 가능합니다. 데이터베이스의 Primary Key로 사용될 예정입니다."

### 6. "90% 통계의 출처는?"
> "Europol 2024 보고서 'Facing Reality'에서 2026년까지 온라인 콘텐츠의 90%가 AI 생성일 수 있다고 예측했습니다."
